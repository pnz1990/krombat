# Kubernetes RPG ‚Äî Technical Details

## Dungeon CR Spec Fields
Every game instance is a Dungeon custom resource. All mutable game state lives in spec fields:

- `monsters` (int) ‚Äî number of monsters
- `difficulty` (string) ‚Äî easy/normal/hard
- `heroClass` (string) ‚Äî warrior/mage/rogue
- `heroHP` (int) ‚Äî current hero hit points
- `heroMana` (int) ‚Äî mage mana (0 for other classes)
- `monsterHP` ([]int) ‚Äî HP array for each monster
- `bossHP` (int) ‚Äî boss hit points
- `modifier` (string) ‚Äî none/curse-fortitude/curse-fury/curse-darkness/blessing-strength/blessing-resilience/blessing-fortune
- `tauntActive` (int) ‚Äî warrior taunt state
- `backstabCooldown` (int) ‚Äî rogue backstab cooldown turns
- `inventory` (string) ‚Äî CSV of item IDs (e.g. "weapon-epic,hppotion-rare")
- `weaponBonus` (int), `weaponUses` (int) ‚Äî equipped weapon
- `armorBonus` (int) ‚Äî equipped armor defense %
- `shieldBonus` (int) ‚Äî equipped shield block %
- `poisonTurns` (int), `burnTurns` (int), `stunTurns` (int) ‚Äî status effects
- `treasureOpened` (int) ‚Äî 0/1
- `doorUnlocked` (int) ‚Äî 0/1
- `currentRoom` (int) ‚Äî 1 or 2
- `room2MonsterHP` ([]int), `room2BossHP` (int) ‚Äî room 2 initial values
- `lastHeroAction` (string) ‚Äî last combat result text
- `lastEnemyAction` (string) ‚Äî last enemy action text
- `lastCombatLog` (string) ‚Äî JSON with full combat details
- `lastLootDrop` (string) ‚Äî item ID of last loot drop (empty if none)
- `attackSeq` (int) ‚Äî monotonic counter incremented by each attack

## Nine ResourceGraphDefinitions

### dungeon-graph (parent)
Creates: Namespace, Hero CR, Monster CRs (forEach), Boss CR, Treasure CR, Modifier CR, GameConfig ConfigMap.
Status fields derived via CEL: livingMonsters, bossState, victory, defeat, loot, treasureState, modifier, modifierType, maxMonsterHP, maxBossHP, maxHeroHP, diceFormula, monsterCounter, bossCounter.

### hero-graph
Hero CR ‚Üí ConfigMap with HP, class, mana, stats. All derived via CEL from Hero spec.

### monster-graph
Monster CR ‚Üí ConfigMap. entityState derived via CEL: `spec.hp > 0 ? 'alive' : 'dead'`.

### boss-graph
Boss CR ‚Üí ConfigMap. entityState derived via CEL from HP and monstersAlive count.

### treasure-graph
Treasure CR ‚Üí ConfigMap + Secret. State: unopened/opened. Key format: `DUNGEONNAME-NN-RUNE` (CEL: upperAscii + size*7).

### modifier-graph
Modifier CR ‚Üí ConfigMap. 6 modifier types with effects computed via CEL.

### loot-graph
Loot CR ‚Üí Secret. Item data: type, rarity, stat, description via CEL.

### attack-graph (combat only)
Attack CR ‚Üí Job. Handles: monster attacks, boss attacks, mage heal, warrior taunt, rogue backstab. Builds comprehensive lastCombatLog JSON. Increments attackSeq. Rejects attacks on dead targets without incrementing attackSeq.

### action-graph (non-combat)
Action CR ‚Üí Job. Handles: equip weapon/armor/shield, use HP/mana potions, open treasure, unlock door, enter room 2. All patches clear lastLootDrop. Enter-room-2 deletes stale attack CRs.

## Attack Job Behavior
- Reads Dungeon CR, computes dice roll, applies modifiers/equipment/class bonuses
- Patches Dungeon CR with merge patch (monsterHP/bossHP, heroHP, status effects, loot, attackSeq)
- Loot drops only on kill transition: OLD_HP > 0 && NEW_HP == 0
- Boss target matching: grep -q "\-boss$" (not just "boss" anywhere)
- Already-dead targets: patches lastHeroAction but does NOT increment attackSeq
- Game-over check: rejects combat when heroHP ‚â§ 0 or (bossHP ‚â§ 0 && all monsters dead)
- Custom job-runner image: alpine + kubectl + jq + bash (~30MB)

## Frontend Architecture
- React SPA with 8-bit pixel art styling
- All state from Dungeon CR spec (not status ‚Äî status can be stale)
- gameOver and bossState derived from spec.bossHP and spec.monsterHP
- Combat poll: reads attackSeq AFTER initial wait, polls for increment
- attackingRef (useRef) prevents double-click submissions
- Item actions early-return (no fallthrough to combat/loot code)
- Dead monsters: 0.35 opacity, no attack button
- Combat modal: attack animation (frames 3-5) for all entities during both rolling and resolved phases
- Sprite system: individual frame images, hurt=6‚Üí1‚Üí6 loop, dead=static frame 6
